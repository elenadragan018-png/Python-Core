Colecții avansate

În Python Start ai lucrat cu liste. Python îți oferă însă o întreagă familie de structuri de date, fiecare cu superputerea ei: tuple (imutabil), dict (perechi cheie–valoare), set (elemente unice). Alegerea structurii potrivite este jumătate din rezolvarea unei probleme.

🎯 La finalul lecției vei putea:

  • să alegi argumentat între listă, tuplu, dicționar și mulțime pentru o problemă dată;
  • să creezi și să parcurgi dicționare (chei, valori, perechi) și mulțimi;
  • să aplici operații pe mulțimi: reuniune, intersecție, diferență;
  • să scrii list/dict/set comprehensions pentru transformări de date pe o singură linie.

2. Tuple — datele care nu se schimbă

Un tuplu este o listă „înghețată”. Odată creat, nu mai poate fi modificat — și exact asta îl face valoros: garantezi că nimeni (nici tu, din greșeală) nu îi schimbă conținutul.

# Un punct pe hartă nu are voie să „alunece” singur
punct = (45.79, 24.15)        # lat, lon — Sibiu
culoare = (37, 99, 235)       # RGB pentru albastrul Python Core

lat, lon = punct              # despachetare (unpacking)
print(lat)                    # 45.79

punct[0] = 46.0               # 💥 TypeError: 'tuple' object does not
                              #    support item assignment
💡 Despachetarea — cel mai elegant truc Python a, b = b, a interschimbă două variabile fără variabilă auxiliară. În spate, Python construiește un tuplu temporar (b, a) și îl despachetează. În alte limbaje ai avea nevoie de 3 linii.

3. Dict — agenda universală

Dicționarul leagă o cheie unică de o valoare. Căutarea după cheie este aproape instantanee, indiferent câte elemente conține — de aceea dict este probabil cea mai folosită structură din tot Python-ul real.

elev = {"nume": "Andrei", "clasa": "X B", "medie": 9.25}

print(elev["medie"])           # 9.25 — acces după cheie
elev["oras"] = "București"     # adăugare
elev["medie"] = 9.50           # modificare

print(elev.get("telefon", "lipsește"))  # nu crapă: "lipsește"

for cheie, valoare in elev.items():     # parcurgere perechi
    print(f"{cheie} → {valoare}")
⚠️ Capcana clasică elev["telefon"] pe o cheie inexistentă produce KeyError și oprește programul. elev.get("telefon") returnează None (sau valoarea implicită pe care i-o dai) și programul merge mai departe. Reține diferența — revine la L8 (excepții) și în testul final.

4. Set — clubul elementelor unice

voturi = ["ana", "dan", "ana", "ioana", "dan", "ana"]
participanti = set(voturi)
print(participanti)            # {'ana', 'dan', 'ioana'} — duplicate eliminate

info = {"ana", "dan", "mihai"}   # elevii de la cercul de informatică
robo = {"dan", "ioana"}          # elevii de la robotică

print(info | robo)             # reuniune:    {'ana','dan','mihai','ioana'}
print(info & robo)             # intersecție: {'dan'}
print(info - robo)             # diferență:   {'ana','mihai'}
print("dan" in info)           # True — test de apartenență instant

5. Comprehensions — fabrica de colecții pe o linie

O comprehension comprimă tiparul „creez o colecție goală + for + append” într-o singură expresie lizibilă. Există pentru liste, dicționare și mulțimi.

# Varianta clasică (Python Start)          # Varianta comprehension (Python Core)
patrate = []
for x in range(6):
    patrate.append(x ** 2)
                                            patrate = [x ** 2 for x in range(6)]
# Ambele: [0, 1, 4, 9, 16, 25]

# Cu filtrare:
pare = [x for x in range(20) if x % 2 == 0]

# Dict comprehension — codul ASCII al fiecărei litere:
coduri = {litera: ord(litera) for litera in "abc"}
# {'a': 97, 'b': 98, 'c': 99}

# Set comprehension — restul împărțirii la 4:
resturi = {x % 4 for x in range(100)}    # {0, 1, 2, 3}
💡 Regula bunului-simț Dacă o comprehension nu mai încape lizibil pe o linie (sau are mai mult de un if și un for), scrie buclă clasică. Codul se scrie o dată, dar se citește de zeci de ori.

Șiruri și expresii regulate

Cea mai mare parte a datelor din lume este text: mesaje, loguri, fișiere CSV, pagini web. Python excelează la prelucrarea textului prin metodele clasei str, prin f-strings și — pentru tipare complexe — prin modulul re (expresii regulate).

🎯 La finalul lecției vei putea:

  • să aplici metodele esențiale ale șirurilor: strip, split, join, replace, lower/upper, startswith, find;
  • să formatezi elegant ieșirea cu f-strings (aliniere, zecimale, mii);
  • să citești și să scrii expresii regulate simple cu re.search și re.findall;
  • să validezi date de intrare (CNP-uri de test, e-mailuri, numere de telefon) cu tipare regex.

1. Metodele șirurilor — trusa de bază

Un șir este imutabil (ca tuplul!): metodele nu modifică șirul original, ci returnează unul nou.

mesaj = "  Buna ziua, Python!  "

mesaj.strip()                  # "Buna ziua, Python!" — taie spațiile de la capete
mesaj.lower()                  # "  buna ziua, python!  "
mesaj.replace("ziua", "seara") # înlocuiește toate aparițiile
mesaj.strip().startswith("Buna")  # True

csv = "Ana,9.50,X B"
campuri = csv.split(",")       # ['Ana', '9.50', 'X B'] — text → listă
" | ".join(campuri)            # "Ana | 9.50 | X B"  — listă → text
💡 split + join = perechea de aur split sparge textul în bucăți, join le lipește la loc. Cu aceste două metode prelucrezi fișiere CSV, loguri de server și liste de cuvinte fără nicio bibliotecă suplimentară.

2. f-strings — formatare modernă

nume, medie, pret = "Ana", 9.4567, 1234567

print(f"Eleva {nume} are media {medie:.2f}")   # 2 zecimale: 9.46
print(f"Preț: {pret:,} lei")                    # separatori de mii: 1,234,567
print(f"{nume:>10}|")                           # aliniere dreapta:  '       Ana|'
print(f"{2 ** 10 = }")                          # debugging: '2 ** 10 = 1024'

3. Expresii regulate — limbajul tiparelor

O expresie regulată (regex) descrie un tipar de text, nu un text exact. În loc să cauți „0722123456”, cauți „zece cifre care încep cu 07”. Modulul standard este re.

import re

text = "Suntem in anul 2026, clasa a 10-a, sala 204."

re.findall(r"\d+", text)        # ['2026', '10', '204'] — toate numerele

rezultat = re.search(r"\d{4}", text)   # primul grup de exact 4 cifre
if rezultat:
    print(rezultat.group())       # '2026'

# Validare număr de telefon românesc:
tipar = r"^07\d{8}$"
print(bool(re.search(tipar, "0722123456")))   # True
print(bool(re.search(tipar, "0722-12-34")))   # False

4. Studiu mini-practic: curățarea unui fișier de note

import re

linii = ["  Ana ; 9,50 ", "Dan;8.75", " Ioana ;10 "]

for linie in linii:
    nume, nota = [camp.strip() for camp in linie.split(";")]
    nota = nota.replace(",", ".")              # 9,50 → 9.50
    if re.search(r"^\d+(\.\d+)?$", nota):       # e număr valid?
        print(f"{nume:<8} → {float(nota):.2f}")
# Ana      → 9.50
# Dan      → 8.75
# Ioana    → 10.00

Funcții avansate

Știi deja să definești funcții cu def. Acum urcăm un nivel: în Python, funcțiile sunt obiecte de primă clasă — pot fi anonime (lambda), pot fi pasate altor funcții (map, filter), se pot apela pe ele însele (recursivitate) și pot fi „împachetate” în alte funcții (decoratori).

🎯 La finalul lecției vei putea:

  • să scrii funcții lambda și să decizi când sunt potrivite (și când nu);
  • să transformi și să filtrezi colecții cu map, filter și sorted(key=...);
  • să analizezi o funcție recursivă: caz de bază, caz recursiv, adâncimea apelurilor;
  • să explici ce face un decorator simplu și să folosești unul existent.

1. Lambda — funcția fără nume

O funcție lambda este o funcție de o singură expresie, definită „din mers”, de obicei ca argument pentru altă funcție.

dublu = lambda x: x * 2          # echivalent cu def dublu(x): return x * 2
print(dublu(21))                  # 42

# Adevărata utilitate: ca parametru `key` la sortare
elevi = [("Ana", 9.5), ("Dan", 8.7), ("Ioana", 10.0)]
elevi_sortati = sorted(elevi, key=lambda e: e[1], reverse=True)
# [('Ioana', 10.0), ('Ana', 9.5), ('Dan', 8.7)] — descrescător după medie
💡 Regula de stil Dacă o lambda are nevoie de nume sau de mai mult de o expresie — scrie def. Lambda este pentru funcții „de unică folosință”, pasate pe loc.

2. map și filter — banda transportoare

note = [9.5, 4.2, 8.75, 3.9, 10.0]

rotunjite = list(map(round, note))                # [10, 4, 9, 4, 10]
promovati = list(filter(lambda n: n >= 5, note))  # [9.5, 8.75, 10.0]

# Echivalentele comprehension — adesea mai lizibile:
rotunjite = [round(n) for n in note]
promovati = [n for n in note if n >= 5]

map aplică o funcție pe fiecare element; filter păstrează doar elementele pentru care funcția returnează True. Ambele returnează obiecte „leneșe” — le convertim cu list() ca să vedem rezultatul.

3. Recursivitatea — funcția care se cheamă singură

O funcție recursivă rezolvă o problemă reducând-o la o versiune mai mică a ei însăși. Are obligatoriu două părți:

  • cazul de bază — situația simplă, rezolvată direct, care oprește lanțul;
  • cazul recursiv — apelul către sine, cu o problemă strict mai mică.
def factorial(n):
    if n <= 1:                # cazul de bază
        return 1
    return n * factorial(n - 1)   # cazul recursiv: problemă mai mică

print(factorial(5))           # 120

# Cum se desfășoară apelurile:
# factorial(5) → 5 * factorial(4)
#                    → 4 * factorial(3)
#                         → 3 * factorial(2)
#                              → 2 * factorial(1)
#                                   → 1        (baza!)
# … apoi totul se „strânge” înapoi: 2·1=2, 3·2=6, 4·6=24, 5·24=120
💥 Ce se întâmplă fără caz de bază? Apelurile nu se opresc niciodată; Python umple stiva de apeluri și ridică RecursionError: maximum recursion depth exceeded (implicit ~1000 de niveluri). Întrebare de acest tip apare frecvent la evaluări — inclusiv în testul final al cursului.
📌 Studiu de caz · ancorare reală

Cum „recurge” Google prin Internet — PageRank simplificat

Algoritmul istoric al căutării Google, PageRank, pornește de la o idee recursivă: o pagină este importantă dacă spre ea trimit pagini importante. Definiția importanței unei pagini depinde de importanța altor pagini — care depinde, la rândul ei, de altele. Exact structura unei funcții recursive: problema se exprimă prin versiuni „mai mici” ale ei însăși, iar calculul se repetă până când valorile se stabilizează (cazul de bază practic: „nu se mai schimbă nimic semnificativ”). Observație de finețe: definiția este recursivă, dar calculul practic de mai jos este iterativ (o buclă care repetă pasul până la stabilizare) — în informatică, o problemă definită recursiv se poate rezolva adesea și iterativ, iar alegerea variantei este o decizie de proiectare.

# PageRank de jucărie: 3 pagini care se leagă între ele
linkuri = {"A": ["B", "C"], "B": ["C"], "C": ["A"]}
rank = {p: 1.0 for p in linkuri}          # toate pornesc egale

for pas in range(20):                      # repetăm până se stabilizează
    nou = {p: 0.15 for p in linkuri}
    for pagina, iesiri in linkuri.items():
        for tinta in iesiri:
            nou[tinta] += 0.85 * rank[pagina] / len(iesiri)
    rank = nou

print({p: round(r, 2) for p, r in rank.items()})
# C iese cea mai „importantă” — primește linkuri de la A și B

Observă cum lecțiile se leagă: dict + comprehension (L1), bucle imbricate (Python Start), gândire recursivă (L3).

4. Decoratori — funcții care îmbracă funcții

Un decorator primește o funcție și returnează o versiune „îmbunătățită” a ei, fără să-i modifice codul. Sintaxa @decorator deasupra unui def este doar zahăr sintactic pentru functie = decorator(functie).

import time

def cronometru(functie):
    def invelis(*args, **kwargs):
        start = time.perf_counter()
        rezultat = functie(*args, **kwargs)     # apelăm originalul
        durata = time.perf_counter() - start
        print(f"{functie.__name__} a durat {durata:.4f}s")
        return rezultat
    return invelis

@cronometru
def suma_mare():
    return sum(range(10_000_000))

suma_mare()      # suma_mare a durat 0.21xx s
💡 Unde îi vei reîntâlni În frameworkuri reale decoratorii sunt peste tot: @app.route("/") în Flask, @staticmethod și @property în OOP (L6–L7), @pytest.fixture la testare. Pentru clasa a X-a e suficient să îi citești corect: „funcția de sub @ a fost împachetată”.

Activitatea 3 · Trasează recursivitatea

Fie funcția:

def mister(n):
    if n == 0:
        return 0
    return n + mister(n - 1)

Pe caiet: (a) desenează lanțul de apeluri pentru mister(4); (b) ce calculează funcția, în general? (c) ce s-ar întâmpla la apelul mister(-1)?

✅ Verifică-te (a) mister(4) → 4 + mister(3) → 3 + mister(2) → 2 + mister(1) → 1 + mister(0) → 0; rezultat 10. (b) Suma 1+2+…+n. (c) n scade la −2, −3, … și nu atinge niciodată 0 → RecursionError: cazul de bază există, dar nu este atins — o lecție subtilă: baza trebuie să fie accesibilă din orice intrare validă, iar intrările invalide trebuie respinse (vezi L8).

📌 Ce trebuie să reții pentru examen

  • lambda = funcție anonimă de o expresie; tipic ca key= la sorted/min/max.
  • map(f, col) transformă fiecare element; filter(f, col) păstrează elementele cu f(x) adevărat.
  • O funcție recursivă are obligatoriu caz de bază + caz recursiv cu problemă mai mică.
  • Lipsa (sau neatingerea) cazului de bază → RecursionError.
  • Decorator = funcție care primește o funcție și returnează o funcție; @d ≡ f = d(f).

OOP — Introducere

Până acum, datele (dict-uri, liste) și funcțiile care le prelucrează au trăit separat. Programarea orientată pe obiecte (OOP) le unește: o clasă descrie împreună datele unui lucru și acțiunile lui. Aproape tot software-ul modern — de la jocuri la Instagram — este scris așa.

🎯 La finalul lecției vei putea:

  • să explici relația clasă ↔ obiect (șablon ↔ exemplar);
  • să definești o clasă cu __init__, atribute și metode;
  • să explici rolul lui self și de ce apare ca prim parametru;
  • să creezi mai multe obiecte din aceeași clasă și să le folosești independent.

1. Clasa = șablonul, obiectul = exemplarul

Gândește-te la formularul de carnet de elev: formularul (gol) este clasa; carnetul tău completat este un obiect (o instanță). Din același formular se completează 30 de carnete diferite — din aceeași clasă se creează oricâte obiecte.

class Elev:
    """Un elev cu nume și un catalog de note."""

    def __init__(self, nume, clasa):
        self.nume = nume          # atribute = datele obiectului
        self.clasa = clasa
        self.note = []

    def adauga_nota(self, nota):  # metode = acțiunile obiectului
        if 1 <= nota <= 10:
            self.note.append(nota)

    def media(self):
        if not self.note:
            return 0
        return sum(self.note) / len(self.note)

# Creăm obiecte (instanțiere) — __init__ rulează automat:
ana = Elev("Ana Popescu", "X B")
dan = Elev("Dan Ionescu", "X A")

ana.adauga_nota(10)
ana.adauga_nota(9)
dan.adauga_nota(7)

print(ana.media())    # 9.5  — fiecare obiect are PROPRIILE date
print(dan.media())    # 7.0

2. Anatomia: __init__ și self

  • __init__ este constructorul: metoda specială apelată automat la crearea obiectului. Aici primesc valori atributele.
  • self este obiectul curent — „eu însumi”. Când scrii ana.media(), Python apelează în culise Elev.media(ana): obiectul tău devine self.
  • self.nume = atribut al obiectului (fiecare obiect are al lui); nume simplu = variabilă locală a metodei, dispare la final.
⚠️ Cele 3 greșeli clasice ale începătorului OOP 1) Uiți self din lista de parametri → TypeError: media() takes 0 positional arguments but 1 was given. 2) Scrii nume = nume în loc de self.nume = nume → atributul nu se salvează. 3) Apelezi Elev.media() pe clasă, nu pe obiect → nu există date.

3. Atribute de clasă vs. atribute de instanță

class Elev:

    def __init__(self, nume):

ana, dan = Elev("Ana"), Elev("Dan")
print(ana.scoala == dan.scoala)        # True — împărtășit
print(ana.nume == dan.nume)            # False — individual

OOP — Avansat

O singură clasă e utilă; o familie de clase e puternică. Astăzi: moștenirea (clase care extind clase), polimorfismul (același apel, comportamente diferite) și metodele speciale __str__ / __repr__ care fac obiectele „prezentabile”.

🎯 La finalul lecției vei putea:

  • să derivezi o clasă-copil dintr-o clasă-părinte și să refolosești cod cu super();
  • să suprascrii (override) o metodă și să explici ce versiune se apelează;
  • să demonstrezi polimorfismul pe o listă de obiecte din clase diferite;
  • să implementezi __str__ și __repr__ și să distingi rolurile lor;
  • să evaluezi când moștenirea este alegerea corectă (relația „este un fel de”).

1. Moștenirea — „este un fel de”

Un Pinguin este un fel de Pasăre; o PasăreCântătoare la fel. Trec de testul „is-a” → moștenire. Clasa-copil primește gratuit tot ce are părintele și adaugă sau modifică ce-i trebuie.

class Animal:                          # clasa-părinte (de bază)
    def __init__(self, nume):
        self.nume = nume

    def saluta(self):
        return f"{self.nume} face un sunet."

class Caine(Animal):                   # clasa-copil (derivată)
    def saluta(self):                  # OVERRIDE — suprascrie versiunea părintelui
        return f"{self.nume} latră: Ham!"

class Pisica(Animal):
    def __init__(self, nume, vieti=9):
        super().__init__(nume)         # refolosește constructorul părintelui
        self.vieti = vieti

    def saluta(self):
        return f"{self.nume} toarce ({self.vieti} vieți)."

rex = Caine("Rex")
print(rex.saluta())       # Rex latră: Ham!   ← versiunea din Caine câștigă
print(isinstance(rex, Animal))  # True — un Caine ESTE un Animal
💡 Regula de căutare a metodelor (MRO) La obiect.metoda(), Python caută metoda întâi în clasa obiectului, apoi urcă spre părinte, bunic etc. Prima găsită câștigă. De aceea override-ul din Caine „acoperă” versiunea din Animal.

2. Polimorfismul — același verb, acțiuni diferite

adapost = [Caine("Rex"), Pisica("Mia"), Caine("Azor"), Animal("Misterios")]

for animal in adapost:
    print(animal.saluta())     # ACELAȘI apel…
# Rex latră: Ham!              # …comportament de Caine
# Mia toarce (9 vieți).        # …comportament de Pisica
# Azor latră: Ham!
# Misterios face un sunet.     # …comportamentul implicit din Animal

Bucla nu știe (și nu trebuie să știe) ce tip exact are fiecare element — fiecare obiect „știe singur” cum să răspundă. Acesta este polimorfismul, iar el face codul extensibil: mâine adaugi clasa Papagal și bucla funcționează neschimbată.

3. __str__ și __repr__ — cartea de vizită a obiectului

class Elev:
    def __init__(self, nume, medie):
        self.nume, self.medie = nume, medie

    def __str__(self):                 # pentru OAMENI — print(), f-strings
        return f"{self.nume} (media {self.medie:.2f})"

    def __repr__(self):                # pentru PROGRAMATORI — debugging, consolă
        return f"Elev(nume={self.nume!r}, medie={self.medie})"

ana = Elev("Ana", 9.5)
print(ana)            # Ana (media 9.50)            ← __str__
print([ana])          # [Elev(nume='Ana', medie=9.5)] ← în colecții se folosește __repr__!

Fără aceste metode, print(ana) afișează inutilul <__main__.Elev object at 0x7f...>. Convenție: __str__ = lizibil pentru utilizator; __repr__ = precis, ideal cod care ar recrea obiectul.

Arhitectura unui joc — de ce orice joc e un exercițiu de OOP

Deschide codul oricărui joc (text sau pygame) și vei găsi același schelet: o ierarhie de entități cu date + comportament. Relația „este un fel de” apare natural: și Player, și Enemy sunt un fel de entitate cu poziție și viață.

class Entitate:
    def __init__(self, nume, viata):
        self.nume, self.viata = nume, viata

    def primeste_lovitura(self, dmg):
        self.viata = max(0, self.viata - dmg)

    def traieste(self):
        return self.viata > 0

class Player(Entitate):
    def __init__(self, nume):
        super().__init__(nume, viata=100)
        self.scor = 0

class Enemy(Entitate):
    def __init__(self, nume, putere):
        super().__init__(nume, viata=30)
        self.putere = putere

class Game:
    def __init__(self, player, inamici):
        self.player, self.inamici = player, inamici

    def runda(self):
        for e in self.inamici:
            if e.traieste():
                self.player.primeste_lovitura(e.putere)   # polimorfism în acțiune

Motorul jocului (Game) lucrează cu Entități, fără să-i pese cine e Player și cine e Enemy. Aceeași arhitectură o vei reîntâlni în pygame, Unity (C#) sau Godot — limbajul se schimbă, ideea nu.

  • Prinde excepții specifice; except: gol = anti-model.
  • raise ValueError("mesaj") — semnalezi tu datele invalide.
  • Traceback-ul se citește de jos în sus: tipul erorii → linia → lanțul de apeluri.
  • logging > print: nivele, timestamp, configurabil.

Proiect aplicativ — Manager de contacte

Momentul sintezei: construiești o mini-aplicație CLI completă care unește tot cursul — OOP (L6–L7), JSON (L4), excepții și logging (L8), colecții (L1) și validare cu regex (L2). Nu mai citești cod: creezi.

🎯 La finalul lecției vei putea:

  • să proiectezi arhitectura unei aplicații mici: clase, responsabilități, fluxul datelor;
  • să implementezi persistența datelor cu JSON (încărcare la pornire, salvare la modificare);
  • să construiești un meniu CLI robust, care nu cade la nicio intrare a utilizatorului;
  • să extinzi singur aplicația cu funcționalități noi (sarcinile de extindere).

1. Specificația aplicației

„Contacte” — un manager de contacte în linie de comandă:

  1. adaugă un contact (nume, telefon validat cu regex, email);
  2. listează toate contactele, sortate alfabetic;
  3. caută un contact după fragment de nume;
  4. șterge un contact;
  5. datele persistă între rulări în contacte.json;
  6. nicio intrare a utilizatorului nu trebuie să poată „omorî” aplicația.

2. Arhitectura — cine ce face

ComponentăResponsabilitateLecția-sursă

3. Codul complet, comentat (~80 de linii)

"""Contacte — manager CLI. Python Core, L9."""
import json
import logging
import re

logging.basicConfig(level=logging.INFO, format="[%(levelname)s] %(message)s")
FISIER = "contacte.json"


class Contact:
    TIPAR_TEL = r"^07\d{8}$"                       # atribut de clasă (L6)

    def __init__(self, nume, telefon, email=""):
        if not nume.strip():
            raise ValueError("Numele nu poate fi gol.")
        if not re.search(self.TIPAR_TEL, telefon):  # regex (L2)
            raise ValueError(f"Telefon invalid: {telefon} (format: 07XXXXXXXX)")
        self.nume, self.telefon, self.email = nume.strip(), telefon, email

    def to_dict(self):                              # obiect → dict, pentru JSON
        return {"nume": self.nume, "telefon": self.telefon, "email": self.email}

    def __str__(self):                              # L7
        return f"{self.nume:<20} {self.telefon}  {self.email}"


class AgendaContacte:
    def __init__(self, fisier=FISIER):
        self.fisier = fisier
        self.contacte = []                          # listă de obiecte Contact
        self._incarca()

    def _incarca(self):
        try:
            with open(self.fisier, encoding="utf-8") as f:
                for d in json.load(f):              # dict → obiect
                    self.contacte.append(Contact(**d))
            logging.info(f"Încărcate {len(self.contacte)} contacte.")
        except FileNotFoundError:
            logging.warning("Fișier inexistent — pornesc cu agendă goală.")
        except (json.JSONDecodeError, ValueError) as e:
            logging.error(f"Date corupte: {e}. Pornesc cu agendă goală.")

    def _salveaza(self):
        with open(self.fisier, "w", encoding="utf-8") as f:
            json.dump([c.to_dict() for c in self.contacte],  # comprehension (L1)
                      f, indent=2, ensure_ascii=False)

    def adauga(self, contact):
        self.contacte.append(contact)
        self._salveaza()

    def listeaza(self):
        return sorted(self.contacte, key=lambda c: c.nume.lower())  # lambda (L3)

    def cauta(self, fragment):
        return [c for c in self.contacte
                if fragment.lower() in c.nume.lower()]

    def sterge(self, nume):
        inainte = len(self.contacte)
        self.contacte = [c for c in self.contacte if c.nume != nume]
        self._salveaza()
        return inainte - len(self.contacte)            # câte am șters


def main():
    agenda = AgendaContacte()
    meniu = "\n[1] Adaugă  [2] Listează  [3] Caută  [4] Șterge  [0] Ieșire\n> "
    while True:
        optiune = input(meniu).strip()
        try:
            if optiune == "1":
                c = Contact(input("Nume: "), input("Telefon: "), input("Email: "))
                agenda.adauga(c)
                print("✓ Salvat.")
            elif optiune == "2":
                for c in agenda.listeaza():
                    print(c)                            # __str__ în acțiune
            elif optiune == "3":
                for c in agenda.cauta(input("Caută: ")):
                    print(c)
            elif optiune == "4":
                n = agenda.sterge(input("Nume exact: "))
                print(f"✓ Șterse: {n}")
            elif optiune == "0":
                break
            else:
                print("Opțiune necunoscută.")
        except ValueError as e:                         # intrări invalide ≠ crash
            print("✗", e)


if __name__ == "__main__":                          # L4
    main()
